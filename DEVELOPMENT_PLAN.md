## 1. Меню улик с галочками и сужение круга подозреваемых

**Цель:** игрок отмечает наблюдаемые признаки; список возможных типов призраков сокращается по правилам исключения (как в детективной механике).

**Контекст (уже в проекте):** типы призраков и их «улики» в смысле ЭМП / УФ / шар / радио уже задаются в **`ghost_abilities.ini`**. Класс `GhostAbilitiesConfig` в `ghost.py` читает секции `ghost_*`, мержит с `[ghost_default]` и отдаёт профиль с полями `amp`, `ultraviolet`, `ghostorb`, `radio`, плюс движение (`can_fly`, `can_walk`, скорости и т.д.). Случайный тип на раунд берётся через `random_profile_name()`. Дублировать те же данные в отдельном JSON не нужно: фильтр подозреваемых должен **опираться на те же профили**, что и игра (парсер INI или экспорт в память при старте).

- [ ] **1.1** Свести канонический список **игровых улик для журнала** с тем, что игрок реально может проверить: минимум совпасть с флагами INI (`amp`, `ultraviolet`, `ghostorb`, `radio`); плюс будущие признаки (охота, соль, проектор…) — добавлять их как новые ключи в `ghost_abilities.ini` и в `GhostAbilitiesConfig.BOOL_KEYS` / логику призрака, а не отдельную таблицу «мимо» INI.
- [ ] **1.2** Расширить **данные в `ghost_abilities.ini`**, если для сужения нужно больше, чем четыре флага: например отдельные поля «даёт ложный ЭМП», «редкий орб», уникальные черты охоты — всё в тех же секциях `ghost_N`, чтобы дизайн оставался в одном файле. Реализовать чтение новых полей в `GhostAbilitiesConfig._parse_section` и использовать **тот же** `get_profile(имя)` при расчёте «кому подходит отмеченное». Заготовки `ghost_8`…`ghost_16` уже есть — заполнять их под геймдизайн, а не плодить второй реестр типов.
- [ ] **1.3** Добавить состояние игры: `discovered_evidence: set[str]` и «не уверен» / «исключено» при необходимости.
- [ ] **1.4** Экран или панель «Журнал»: список чекбоксов по уликам; клик переключает «отмечено / не отмечено».
- [ ] **1.5** Алгоритм сужения: обойти **все профили из `GhostAbilitiesConfig.profiles`** (кроме служебного `default`), для каждого взять merge через `get_profile(name)` и отфильтровать по отмеченным уликам; не хардкодить списки имён в коде журнала.
- [ ] **1.6** UI обратной связи: показать «Осталось подозреваемых: N», портреты/имена, подсветка исключённых.
- [ ] **1.7** Горячая клавиша (например `J` или `Tab`) для открытия журнала без паузы или с паузой — решить по геймдизайну.
- [ ] **1.8** Сохранение отмеченных улик в `save.json` (в слот), чтобы не терять прогресс.
- [ ] **1.9** Опционально: кнопка «Сбросить догадки» и защита от случайных кликов (подтверждение).

## 5. Улучшение визуала (конкретные предложения)

### 5.0 Главный экран игры: убрать «простыню из букв»

**Как сейчас:** в `draw_game` (`draws.py`) на игровом виде одновременно рисуются, среди прочего: деньги / уровень / жизни строками слева; блок из **пяти** строк расходников («Аккумуляторы», «Кровь», «Кресты», «Пыль», «Соль»); строка хоткеев `R: радио  E: ЭМП…`; состояние УФ; подписи под слотами инвентаря; кнопки меню. В сумме левая колонка превращается в чтение текста во время напряжённого геймплея — карта и атмосфера теряются.

**Принципы переделки:**

- **Сначала иконки и цифры, текст по требованию.** Постоянно на экране — только то, без чего нельзя (например HP, активный предмет). Названия предметов и подсказки — по наведению, удержанию `Alt` или в отдельной компактной панели.
- **Один визуальный блок HUD**, а не семь разрозненных строк слева: например полупрозрачная «плашка» внизу или вверху с выровненной сеткой иконок.
- **Диэджетика где уместно:** подсказка у компьютера — иконка курсора / «E» на объекте, а не большая реплика над ПК каждый кадр (или показывать один раз при первом входе в зону).
- **Иерархия:** крупно только угроза (охота, мало HP); экономика (деньги) — вторичным размером или в меню паузы / у компьютера.

Задачи:

- [ ] **5.0.1** Собрать **единый модуль отрисовки HUD** (функции или маленький класс): отступы, фон плашки, z-order (мир → затемнение → HUD → модалки).
- [ ] **5.0.2** Заменить пять отдельных строк расходников на **ряд маленьких иконок** (те же `inventory_images`, где есть) + **цифра счётчика** в углу иконки; без длинных подписей «Аккумуляторы:».
- [ ] **5.0.3** Строку хоткеев убрать с постоянного HUD: вынести в **экран «Управление»** в главном меню / настройках и один раз показать туториал; в бою — при первом получении предмета короткий toast («Радио на [R]»).
- [ ] **5.0.4** Подписи под кругами инвентаря: оставить только при **hover** мыши по слоту или показывать **одну** строку «подсказка для выбранного слота» внизу плашки.
- [ ] **5.0.5** Деньги / уровень: компактная строка справа вверху или только **иконка монеты + число**; «Уровень:» — в паузе или у компьютера, если не критичен в бою.
- [ ] **5.0.6** Жизни: **сердечки / полоска** вместо текста «Жизни: N»; критическое состояние — пульс анимации, не новая строка текста.
- [ ] **5.0.7** УФ: индикатор **иконкой** (лампа с подсветкой) + лёгкое изменение оттенка экрана в режиме УФ; убрать отдельную строку «УФ: вкл/выкл», если состояние очевидно визуально.
- [ ] **5.0.8** Проверить **безопасные зоны**: HUD не должен закрывать центр, где игрок и призрак; при `MAP_SCALE` и камере — привязка HUD к краям **окна**, не мира.

### 5.1–5.8 Остальная полировка

- [ ] **5.1** **Журнал улик:** отдельный полноэкранный/модальный экран с пергаментной текстурой, чекбоксы с анимацией; список подозреваемых — **иконки/силуэты** типов + `display_name` из INI, исключённые — серые/зачёркнутые; не дублировать длинные описания на главном HUD.
- [ ] **5.2** **Радио:** при активации — лёгкая виньетка, индикатор «настройка частоты», полоска шума; ответ — **короткая** строка в рамке (1–2 строки), без простыни; опционально субтитры вкл/выкл.
- [ ] **5.3** **Охота:** красная кайма экрана, пульсация, предупреждение за N секунд до фазы (связь с п. 3); при необходимости одно крупное слово-предупреждение вместо абзаца.
- [ ] **5.4** **Следы и УФ:** единый стиль частиц; короткий «остаточный след» на полу после призрака; УФ-режим — не только текстовый флаг, а заметный цветовой слой.
- [ ] **5.5** **Комнаты:** тонкая плашка с названием комнаты (иконка дома + короткое имя), без дублирования с радио-текстом на том же кадре.
- [ ] **5.6** **Шрифты:** один основной + один акцентный; **меньше** мелкого текста 16–22 px на бою; крупные цифры только для урона/таймеров.
- [ ] **5.7** **Магазин:** карточки товара — иконка слева, цена и название, описание в зоне без перекрытия кнопок (текущая двухколоночная схема — база).
- [ ] **5.8** **Пауза / достижения:** toast-уведомления снизу с очередью, иконка + одна строка; не перекрывать слоты инвентаря и центр экрана.

## 7. Что сделать, чтобы в игру было интересно заходить снова

Краткий чеклист «продуктовой» зрелости:

- [ ] **7.1** **Петля раунда:** заход → сбор улик → сужение → риск (охота) → награда (деньги, ачивки) → апгрейд в магазине → следующий раунд.
- [ ] **7.2** **Вариативность:** несколько типов призраков с реально разным поведением (не только текст).
- [ ] **7.3** **Честная обратная связь:** игрок понимает, *почему* его наказали/наградили.
- [ ] **7.4** **Сессия 10–20 минут:** сохранение, быстрый рестарт, понятный прогресс за вечер.
- [ ] **7.5** **Полировка:** звуки шагов, дверей, радио, низкая громкость фона; настройки громкости уже или в плане.
- [ ] **7.6** **Контент:** второй уровень или процедурная расстановка улик на одной карте.
- [ ] **7.7** **Онбординг:** первый заход с подсказками на экране, без стены текста.
