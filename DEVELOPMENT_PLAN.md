# Development Plan

План отражает текущее состояние проекта после появления цепочки уровней: `level_1 -> level_2`. Главная цель ближайших итераций — сделать короткую, понятную кампанию: расследование, победа, переход на следующий уровень, сохранение прогресса и постепенная полировка интерфейса.

---

## 3. Чеклист добавления нового уровня

1. Создать файл `levelN.json` рядом с `level1.json` и `level2.json`.
2. В JSON обязательно добавить `background`, `walls`, `hitboxes`, `computer`, `ghost_spawns`, `rooms`.
3. Проверить, что координаты указаны в базовом размере карты, без ручного умножения на `MAP_SCALE`.
4. Добавить уровень в `levels_index.json`:
   - `id`: например `level_3`;
   - `name`: название для UI;
   - `file`: например `level3.json`;
   - `number`: следующий номер;
   - `background_color_index`: индекс оттенка фона;
   - `next`: id следующего уровня или `null`, если это финал.
5. В предыдущем уровне в `levels_index.json` поставить `"next": "level_N"`.
6. Запустить игру, пройти предыдущий уровень и проверить кнопку «Следующий уровень».
7. Проверить, что на новом уровне появляются призраки, работает фонарик, комнаты затемняются корректно, компьютер доступен.
8. Проверить сохранение: выйти в меню с нового уровня, загрузить слот и убедиться, что загружается нужный номер уровня.
9. Обновить этот план: отметить статус нового уровня и записать задачи по балансу.
