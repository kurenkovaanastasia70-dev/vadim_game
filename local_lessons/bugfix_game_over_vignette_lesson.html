Теория: один источник правды

В UI-баге инвентаря проблема была не только в картинках. Отрисовка стала выпадающим списком, а клики всё ещё проверялись по старым кругам. Это два разных источника правды.

cx, cy, radius = mechanics.inventory_slot_screen(i)
if distance(mouse, (cx, cy)) <= radius:
    use_item_by_index(i)

Правильный подход: если hit-test использует круги, значит отрисовка должна использовать ту же функцию геометрии. Поэтому круги вернулись в `draw_game()`, а `handlers.py` остался синхронизированным с ними.

Баг 1: картинки инвентаря смещены

Симптом: ассеты имеют прозрачные поля, а код растягивал весь файл до квадрата. В итоге реальный предмет внутри PNG оказывался не по центру круга.

Фикс: перед масштабированием берём alpha-bounding-box, обрезаем прозрачные поля, сохраняем пропорции и центрируем на квадратном canvas.

bounds = img.get_bounding_rect()
cropped = img.subsurface(bounds).copy()
canvas.blit(fitted, fitted.get_rect(center=(size // 2, size // 2)))

Такой helper используется и для иконок магазина, и для нижних кругов инвентаря.

Баг 2: расходники и покупка

Симптом: `game.inventory` отвечал сразу за два смысла: предмет куплен и предмет есть в наличии. Для соли, крови, креста, батарейки и пыли это разные вещи.

Фикс: инструменты проверяются по bool-флагу покупки, а расходники по `item_counts`. Покупка расходников идёт через единый `buy_item()`, который увеличивает счётчик.

if item_type in item_counts:
    return get_count(item_type) > 0
return game.inventory.get(name, False)

Баг 3: журнал не ставил игру на паузу

Симптом: ввод игрока блокировался, но `ghost_manager.update()`, урон, проектор и ловушки продолжали обновляться.

Фикс: добавлен единый метод `is_gameplay_paused()`. Все gameplay-тики проверяют его перед обновлением.

if state == GAME and hp > 0 and not is_gameplay_paused():
    update_placed_items()
    update_projector()
    tick_hunt_timer()
    ghost_manager.update(...)

Теперь журнал работает как пауза: игрок может спокойно отмечать улики.

Баг 4: радио без таймера охоты

Симптом: радио отвечало про комнату и настроение призрака, но не сообщало время до охоты.

Фикс: добавлен runtime-таймер охоты. Радио дописывает строку `До охоты: MM:SS`, а во время активной охоты сообщает, что она началась.

ok, text = ghost_manager.ask_radio(player_rect)
text = f"{text} {game.get_hunt_radio_text(ok)}"

Таймер уменьшается только при активном gameplay, поэтому открытый журнал его не ускоряет.

Баг 5: размер предметов на карте

Симптом: один `PLACED_ITEM_SIZE` задавал и хитбокс, и размер картинки. Любое визуальное изменение могло сломать коллизии.

Фикс: хитбокс оставлен отдельной константой, а картинки соли, пыли и проектора масштабируются отдельно и рисуются по центру.

PLACED_ITEM_HITBOX_SIZE = 50
sprite_rect = sprite.get_rect(center=(x - camera_x, y - camera_y))

Баг 6: сохранение размещённых предметов

Симптом: при выходе в меню расходники уже потрачены, но поставленные на карту объекты не сохранялись.

Фикс: `InventoryManager` получил методы сериализации и восстановления runtime-состояния: `placed_items`, `placed_projector`, заряд и состояние срабатывания.

{
  "placed_items": [{"type": "SALT", "x": 420, "y": 510, "triggered": false}],
  "placed_projector": {"x": 600, "y": 380, "powered": true}
}

Баг 7: конфиг призраков

INI-файл `ghost_abilities.ini` уже передавался в `GhostManager`, но случайный выбор мог взять заготовки `ghost_8`-`ghost_16`, которых нет в списке журнала.

Фикс: случайный выбор ограничен полностью описанными профилями `1`-`7`, чтобы игровой тип совпадал с типами расследования.

Баг 8: неверный выбор призрака

Симптом: журнал только фильтровал кандидатов, но не давал финального риска. Игрок мог отмечать улики, но выбор конкретного типа не завершал расследование.

Фикс: строки кандидатов стали кликабельными. Если выбранный тип не совпадает с реальным `ghost_kind`, игра переходит в `GAME_OVER` с причиной `wrong_ghost`.

if profile_id == actual_ghost_kind:
    show_info("Призрак определён верно.")
else:
    enter_game_over(reason="wrong_ghost")

Экран проигрыша теперь смотрит на причину: при `wrong_ghost` показывает текст “Вы выбрали неверного призрака”, а при обычном уроне оставляет сообщение про HP.

Контрольные вопросы

  1. Почему нельзя рисовать инвентарь одним способом, а клики проверять другим?
  2. Чем отличается “предмет куплен” от “расходник есть в количестве больше нуля”?
  3. Какие обновления должны остановиться при открытом журнале?
  4. Почему таймер охоты нельзя уменьшать во время паузы?
  5. Зачем разделять визуальный размер предмета и его хитбокс?
  6. Какие данные нужны, чтобы восстановить поставленный проектор после загрузки?
  7. Почему заготовочные профили призраков опасно выбирать случайно?
  8. Почему для неверного выбора призрака лучше хранить отдельную причину game over, а не менять общий текст навсегда?

Практика

Мини-чек-лист

  • Купить все предметы и проверить нижние прозрачные круги.
  • Потратить соль до нуля и убедиться, что она исчезла из активного инвентаря.
  • Открыть журнал рядом с призраком и проверить, что HP и проектор не меняются.
  • Нажать радио и увидеть время до охоты.
  • Поставить соль, пыль, проектор, сохраниться и загрузиться.
  • Кликнуть неверного кандидата в журнале и проверить текст “Вы выбрали неверного призрака”.
