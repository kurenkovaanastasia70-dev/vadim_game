# План урока: API и Google Sheets для игры (60 минут)

## Цель
Научиться подключать игру к внешним данным через API и понять, как безопасно и удобно обновлять контент без правки файлов игры.

## Результат урока
1. Ученик понимает, что такое API, endpoint, запрос, ответ, формат JSON/CSV.
2. Ученик умеет создать Google Sheet как таблицу данных для игры.
3. Ученик умеет получить CSV-URL и подключить его через переменную окружения.
4. Ученик видит fallback-логику: если сеть недоступна, игра берёт локальный CSV.

## Тайминг (60 минут)

### 0-7 мин: Разминка и мотивация
1. Что неудобно в ручной правке `save.json` и коде.
2. Почему API полезен в играх: контент меняется без сборки.

### 7-18 мин: Теория API (база)
1. API = договор между двумя программами.
2. Термины:
- `client` (игра)
- `server` (Google)
- `endpoint` (URL)
- `request` (запрос)
- `response` (ответ)
- `status code` (200, 404, 500)
3. Форматы данных:
- JSON
- CSV (удобно для таблиц)

### 18-28 мин: Google Sheets как API-источник
1. Структура таблицы ачивок:
- `id,title,description,event_key,target,reward`
2. Публикация в интернет.
3. Export URL:
`https://docs.google.com/spreadsheets/d/<ID>/export?format=csv&gid=<GID>`

### 28-38 мин: Интеграция в игре
1. Где в проекте это подключено: `progression.py`.
2. Провайдеры:
- `GoogleSheetsAchievementTableProvider`
- `LocalAchievementTableProvider`
3. Fallback:
- если URL/сеть не работает -> локальный CSV.

### 38-48 мин: Практика ученика
1. Добавить 1 новую ачивку в таблицу.
2. Проверить, что игра подтягивает её.
3. Прочитать лог в консоли:
- `подтянулась таблица достижений: ...`

### 48-55 мин: Ошибки и отладка
1. Частые проблемы:
- Неправильный `gid`
- Таблица не опубликована
- Пустые/нечисловые `target/reward`
2. Как проверять: открыть export URL в браузере.

### 55-60 мин: Закрепление
1. Мини-квиз по API.
2. Домашка: сделать отдельную таблицу для `tasks`.

## Теория API (подробно)

### Как работает запрос
1. Игра формирует URL.
2. Отправляет HTTP GET.
3. Получает текст CSV.
4. Парсит строки в объекты.
5. Валидирует поля (`target`, `reward` как числа).
6. Передаёт в игровой менеджер прогресса.

### Почему нужен fallback
1. Интернет может пропасть.
2. Google может вернуть ошибку/лимит.
3. Таблица может быть временно пустой.
4. Игра должна продолжать работать офлайн.

### Что важно по безопасности
1. Для урока используем только public read-only CSV.
2. Не хранить приватные ключи в репозитории.
3. Если позже нужен приватный доступ — отдельный backend.

## Команды для запуска с таблицей
```powershell
$env:GOOGLE_SHEETS_ACHIEVEMENTS_CSV_URL="https://docs.google.com/spreadsheets/d/<ID>/export?format=csv&gid=<GID>"
python main_work.py
```

## Чеклист урока
1. Таблица опубликована.
2. Export URL открывается в браузере.
3. Игра стартует без ошибок.
4. В консоли есть лог источника таблицы.
5. Новая ачивка из таблицы видна в игре.
