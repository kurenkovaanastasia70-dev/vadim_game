# Архитектура приложения

Документ описывает текущую структуру проекта, где зашита логика, и какие механики уже реализованы.

## 1) Структура модулей

- `main_work.py` — главный класс `Game`, цикл приложения, переключение состояний, загрузка уровней, сохранения, базовая оркестрация.
- `gamestate.py` — enum состояний экранов (меню, игра, магазин, настройки, сложность, сохранения).
- `handlers.py` — обработчики событий (`pygame`) по состояниям.
- `draws.py` — отрисовка всех экранов и HUD.
- `mechanics.py` — движение игрока, коллизии, загрузка/масштабирование данных уровня, границы карты.
- `assets.py` — загрузка и масштабирование всех ресурсов (спрайты, фон, иконки).
- `inventory_system.py` — предметы, расходники, размещаемые объекты, проектор, эффекты предметов.
- `ghost.py` — ИИ призрака (FSM + A*), улики (ЭМП, следы), менеджер призраков.
- `level_config.py` + `levels_index.json` — реестр уровней и сопоставление id/номеров уровня файлам.
- `button.py` — кнопки UI (`Button`, `PinButton`).
- `level_editor.py` — отдельный графический редактор уровней.

## 2) Основной поток выполнения

1. Вход: `main_work.py` -> создается `Game`.
2. Инициализация:
   - экран и таймер,
   - UI-кнопки,
   - игрок, инвентарь, призраки,
   - загрузка ассетов.
3. Игровой цикл `Game.run()`:
   - `handlers.handle_event(self)` — входные события,
   - в состоянии игры обновляются размещенные предметы/проектор,
   - `self.draw()` — отрисовка текущего состояния,
   - `clock.tick(FPS)`.

## 3) Что делает `Game` (`main_work.py`)

Ключевые методы:

- Состояния и навигация:
  - `set_state()`, `push_state()`, `go_back()`, `change_state()`.
- Уровни:
  - `load_level()`,
  - `load_level_by_id()`,
  - `load_level_for_current_level()`.
- Камера/мир:
  - `update_camera()`.
- Видимость/атмосфера:
  - `_create_vignette_texture()`,
  - `_create_clipped_vignette_overlay()`,
  - `_create_room_visibility_overlay()`.
- Игрок и прогресс:
  - `reset_player_position()`, `reset_for_new_game()`, `buy_item()`.
- Сохранения:
  - `load_saves()`, `save_game()`, `load_game()`, `delete_save()`.
- Отрисовка/цикл:
  - `draw()`, `run()`.

## 4) События и экраны

### `handlers.py`

- `handle_menu_events()` — меню.
- `handle_difficulty_events()` — выбор сложности.
- `handle_saves_events()` — слоты сохранений/удаление/новая игра.
- `handle_shop_events()` — покупки в магазине.
- `handle_settings_events()` — настройки.
- `handle_game_events()` — события игры (инвентарь, клавиши предметов, магазин через компьютер, диалог сохранения).
- `handle_event()` — роутинг в обработчик по `game.state`.

### `draws.py`

- `draw_menu()`, `draw_shop()`, `draw_settings()`, `draw_difficulty()`, `draw_saves()`.
- `draw_game()`:
  - фон с учетом камеры,
  - стены/объекты мира,
  - игрок и текущий кадр спрайта,
  - призрак и улики,
  - затемнение/видимость,
  - HUD и инвентарь.

## 5) Движение, карта, коллизии (`mechanics.py`)

- `update_player_movement(game)`:
  - обработка направлений,
  - движение по осям с проверками,
  - коллизии: стены, хитбоксы, компьютер,
  - обновление `near_computer`,
  - обновление камеры.
- `check_collision*()` — проверки пересечений.
- `inventory_slot_screen(index)` — единая геометрия слотов HUD-инвентаря (используется и в `draws.py`, и в `handlers.py`).
- `load_level_from_json()`, `scale_level_data()`, `generate_walls(level_data)`, `add_map_boundary_walls()`, `get_*_from_level()`.

## 6) Ассеты и масштабирование (`assets.py`)

- Фон уровня: `load_level_background()`.
- Игрок: `load_player_sprites()`.
- Призрак: `load_ghost_sprite()`.
- UI/меню: `load_cork_board()`, `load_pin_images()`, `load_trash_icon()`.
- Инвентарь и размещаемые предметы:
  - `load_inventory_images()`,
  - `load_placement_sprites()`,
  - `load_projector_sprite()`.

## 7) Инвентарь и предметы (`inventory_system.py`)

### Типы и базовые классы

- `ItemType` — перечисление предметов.
- `Item` — базовый абстрактный класс.

### Реализованные предметы

- Фонарь (`Flashlight`),
- Радио (`Radio`),
- ЭМП (`EmfDetector`),
- УФ-фонарь (`UVFlashlight`),
- Аккумулятор (`Battery`),
- Кровь (`Blood`),
- Проектор (`Projector` + `PlacedProjector`),
- Крест (`Cross`),
- Красная пыль (`RedDust`) и соль (`Salt`) через `PlaceableItem` + `PlacedItem`.

### Менеджер инвентаря

`InventoryManager`:
- хранит counts расходников,
- активирует/использует предметы,
- режим размещения предметов на карте,
- рендер размещенных объектов и превью,
- проверяет коллизии призрак <-> размещенные предметы,
- питает проектор и отдает его зоны (`get_projector_zones()`).

## 8) Призрак и ИИ (`ghost.py`)

### Конфиг и служебные сущности

- `GhostAbilitiesConfig` — читает `ghost_abilities.ini`.
- `Node`, `AStar` — поиск пути.
- `Room` — модель комнаты.
- `GhostState` — FSM состояния.

### Состояния (FSM)

`Ghost` использует состояния:
- `INVISIBLE`,
- `IDLE`,
- `PATROL`,
- `WANDER`,
- `SEARCH`,
- `CHASE`,
- `RETURN`.

Ключевые методы:
- `update_state()` — переходы FSM,
- `update_movement()` — движение по состоянию,
- `set_target_with_pathfinding()`/`move_along_path()` — A* и следование пути,
- проверки и столкновения.

### Менеджер призраков

`GhostManager`:
- спавнит и обновляет призрака,
- рисует призрака и debug-комнаты,
- генерирует следы/ЭМП-события,
- API для предметов (`ask_radio()`, `scan_emf()`),
- проверка коллизии с игроком.

## 9) Уже реализованные механики

- Состояния игры и стек переходов.
- Магазин, покупка предметов.
- Система сохранений (3 слота).
- Игровой мир с масштабом карты (`MAP_SCALE`) и камерой.
- Ограничение карты границами.
- Коллизии с:
  - стенами,
  - хитбоксами,
  - компьютером.
- Анимация игрока по направлениям.
- Инвентарь:
  - расходники,
  - предметы-переключатели,
  - размещаемые предметы.
- Проектор:
  - размещение,
  - питание аккумулятором,
  - зона влияния.
- Призрак:
  - FSM,
  - A*,
  - улики (следы, ЭМП),
  - взаимодействие с радио/ЭМП.
- Система видимости/затемнения:
  - режим с фонариком (комната),
  - режим без фонарика (локальный круг).

## 10) `level_editor.py`: назначение и как использовать

`level_editor.py` — отдельное приложение на `pygame` + `tkinter` для подготовки JSON-уровней.

### Что умеет редактор

- Загружать фон.
- Добавлять и редактировать:
  - стены,
  - хитбоксы,
  - компьютер,
  - точки спавна призрака,
  - комнаты.
- Режимы:
  - перемещение объектов,
  - изменение размеров (через углы),
  - удаление выбранного,
  - очистка уровня.
- Привязка к сетке.
- Сохранение/загрузка уровня в JSON.

### Основные методы редактора

- Данные и создание:
  - `add_wall()`, `add_hitbox()`, `set_computer()`, `add_ghost_spawn()`, `add_room()`.
- Выбор/поиск:
  - `get_wall_at_pos()`, `get_hitbox_at_pos()`, `get_computer_at_pos()`, `get_ghost_spawn_at_pos()`, `get_room_at_pos()`, `get_resize_corner()`.
- Управление:
  - `delete_selected()`, `clear_all()`,
  - `save_level()`, `load_level()`,
  - `handle_events()`, `draw()`, `run()`.

### Важные замечания по редактору

- Редактор работает в экранных координатах базового размера (`SCREEN_WIDTH`, `SCREEN_HEIGHT`).
- В игре данные уровня дополнительно масштабируются при загрузке через `scale_level_data()`.
- Верхняя зона (`TOP_BARRIER`) визуализируется как недоступная рабочая область.

## 11) Зависимости и запуск

- Установка:
  - `pip install -r requirements.txt`
- Игра:
  - `python main_work.py`
- Редактор:
  - `python level_editor.py`
